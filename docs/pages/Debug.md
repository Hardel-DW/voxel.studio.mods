### Debug Module
Everything related to Studio's pre-startup uses the Minecraft Logger, as we need to be able to debug startup if it doesn't launch.

Accessible at the bottom right of the sidebar, provides access to various debug options.
- Workspace Page -> Displays all data and the current state of the workspace at time T.
- Logs Page -> Displays application logs. We use a generic API not tied to Compose, which we use for logging throughout the application. (Does not include Network)
- Network Page -> Displays network logs. We hook directly where client/server requests are sent, and make it as generic as possible by filtering by our namespace. I18n with translation keys via dynamic concatenation by identifier, with title and description.
- Code Page -> Displays a readonly code block, a GitHub-style diff block, and a simplified code editor.
- Render Page -> Displays atlas rendering, including the 2D Items atlas generated by Studio.
